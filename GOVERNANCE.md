# Governance

This skill follows the vibecode-prompt-builder family governance model.

## Update policy
Updates are surfaced at the start of each session via the version-check protocol
(see `references/version-check.md`). The skill never updates itself silently.

## Improvement candidates
Shared-process improvements belong in `references/vibecode-core.md`.
emergent.sh-specific improvements belong in `references/platform-emergent.md`.
Neither file may be updated by the skill autonomously.

## Scope
This skill generates prompt artifacts for the user to paste into emergent.sh.
It does not send requests, execute code, or install packages on behalf of the user.
